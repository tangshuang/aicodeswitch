## 1. 功能概述

允许 AICodeSwitch 节点在局域网内发现并同步其他节点的 **Skills** 和 **MCP** 配置。同时支持将远端节点作为供应商导入（可选）。

### 1.1 核心价值

- **Skills 和 MCP 快速同步**：局域网内节点间一键同步 Skills 和 MCP 配置，免重复手动配置
- **可选供应商接入**：远端节点可选择性作为本地供应商，实现代理链路复用
- **安全可控**：同步开关由节点主人自主控制，敏感信息不传输

### 1.2 优先级

| 优先级 | 功能模块 | 说明 |
|--------|---------|------|
| **P0 核心** | Skills 同步 | 局域网内同步远端节点的 Skills |
| **P0 核心** | MCP 同步 | 局域网内同步远端节点的 MCP 配置 |
| P1 次要 | 供应商导入 | 将远端节点作为本地供应商（可选） |

### 1.3 关键约束

- 仅限局域网内使用
- 远端节点的上游 API Key 等敏感信息**不传输**
- 本地 API 服务的 apiKey 需用户**手动填写**（选填）
- Skills / MCP 重名项**禁用且不可勾选**，而非自动跳过

## 2. 用户流程

### 2.1 整体流程图

```

│                    节点 B（本地，拉取方）                          │
│                                                                 │
│  路由管理页面 → 点击"同步配置"按钮                                │
│       │                                                         │
│       ▼                                                         │
│  Step 1：扫描发现 → 选择目标节点                                  │
│       │                                                         │
│       ▼                                                         │
│  Step 2：勾选需要同步的 Skills（重名项禁用）                      │
│       │                                                         │
│       ▼                                                         │
│  Step 3：勾选需要同步的 MCP（重名项禁用）                         │
│       │                                                         │
│       ▼                                                         │
│  Step 4：选择是否将远端作为供应商（可选，可输入 API Key）         │
│       │                                                         │
│       ▼                                                         │
│  Step 5：预览全部选择 → 点击"同步"→ 写入本地 → 自动关闭弹窗      │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 详细步骤

#### Step 0：远端节点开启同步（设置页面）

1. 用户进入设置页面
2. 在现有设置卡片中新增"局域网同步"卡片区域
3. 开启"允许局域网拉取配置"开关
4. 系统自动保存配置，`/api/lan/discover` 接口变为可访问状态

#### Step 1：扫描发现节点

1. 用户在路由管理页面点击"同步配置"按钮，弹窗打开
2. 系统自动开始局域网扫描
3. 并发请求子网内所有 IP 的默认端口 + `/api/lan/discover` 路径
4. 成功响应的节点加入已发现列表
5. 用户从列表中**单选**一个目标节点，点击"下一步"

#### Step 2：选择同步 Skills

1. 系统展示远端节点所有的 Skills 列表
2. 每项显示：名称、类型、描述
3. 用户通过复选框**勾选**需要同步的 Skills
4. **重名检测**：如果本地已存在同名 Skill：
   - 该项复选框**禁用**（disabled），不可勾选
   - 下方显示橙色小字提醒：`"本地已存在，无法重复同步"`
5. 点击"下一步"进入 MCP 选择

#### Step 3：选择同步 MCP

1. 系统展示远端节点所有的 MCP 列表
2. 每项显示：名称、类型（stdio/http/sse）、描述
3. 用户通过复选框**勾选**需要同步的 MCP
4. **重名检测**：如果本地已存在同名 MCP：
   - 该项复选框**禁用**（disabled），不可勾选
   - 下方显示橙色小字提醒：`"本地已存在，无法重复同步"`
5. 点击"下一步"进入供应商配置

#### Step 4：供应商配置（可选）

1. 系统询问用户是否将该远端节点作为本地供应商
2. 选项为"是"/"否"，默认"否"
3. 如果选择"是"：
   - 下方展开一个 API Key 输入框
   - API Key 为**选填**（远端可能未开启认证）
   - 如果不填，创建的本地服务 apiKey 字段为空
4. 点击"下一步"进入最终预览

#### Step 5：预览与确认同步

1. 系统展示完整的同步预览，包含：
   - Skills：即将同步 N 个（列出勾选项）
   - MCP：即将同步 N 个（列出勾选项）
   - 供应商：是否创建（如创建，展示名称和 API Key 填写状态）
2. **所有前置步骤均不写入数据库**，仅在此时确认后写入
3. 用户点击"同步"按钮
4. 弹窗显示 Loading 效果
5. 写入完成后弹窗**自动关闭**

## 3. API 设计

### 3.1 远端节点接口

#### `GET /api/lan/discover`

**不受现有鉴权中间件保护**，其可访问性完全由设置开关控制。

**成功响应（200）**：

```json
{
  "node": {
    "name": "My-AICodeSwitch",
    "version": "1.2.0",
    "port": 4567
  },
  "skills": [
    {
      "id": "skill-1",
      "name": "My Skill",
      "type": "claude-code",
      "description": "..."
    }
  ],
  "mcps": [
    {
      "id": "mcp-1",
      "name": "Vision",
      "type": "stdio",
      "description": "...",
      "command": "npx",
      "args": ["-y", "@anthropic/vision"],
      "env": {}
    }
  ],
  "vendors": [
    {
      "id": "vendor-1",
      "name": "OpenAI",
      "services": [
        {
          "id": "svc-1",
          "name": "GPT-4o",
          "sourceType": "openai-chat",
          "supportedModels": ["gpt-4o", "gpt-4o-mini"],
          "authType": "AUTH_TOKEN"
        }
      ]
    }
  ]
}
```

**拒绝响应（开关关闭时）**：

- 返回 `404 Not Found`，不暴露任何信息
- 不区分"开关关闭"和"服务不存在"

**数据脱敏规则**：

| 字段 | 处理方式 |
|------|---------|
| `skills` 全部字段 | 原样返回（Skills 不含敏感信息） |
| `mcp` 的 command/args/env/url | 原样返回（本地需完整配置才能运行） |
| `mcp` 的 headers | 检查是否含敏感字段（如含 authorization），如有则脱敏 |
| `services.apiKey` | **不返回** |
| `services.apiUrl` | **不返回**（本地会使用远端代理地址代替） |
| `vendors.apiKey` | **不返回** |
| `vendors.apiBaseUrl` | **不返回** |

### 3.2 本地节点接口

#### `GET /api/lan/scan`

前端发起扫描的辅助接口，用于获取本机 IP 和子网信息。

**响应（200）**：

```json
{
  "localIp": "192.168.1.50",
  "subnet": "192.168.1",
  "port": 4567
}
```

#### `POST /api/lan/sync`

执行同步写入操作（仅在前端确认后调用一次）。

**请求体**：

```json
{
  "remoteNode": {
    "ip": "192.168.1.100",
    "port": 4567,
    "name": "My-AICodeSwitch"
  },
  "skills": [
    { "name": "Skill A", "type": "claude-code", "description": "..." }
  ],
  "mcps": [
    { "name": "Vision", "type": "stdio", "command": "npx", "args": ["..."], "env": {} }
  ],
  "vendor": {
    "enabled": true,
    "apiKey": "sk-xxx"
  }
}
```

- `skills`：用户勾选的 Skills 列表（已排除重名项）
- `mcps`：用户勾选的 MCP 列表（已排除重名项）
- `vendor.enabled`：是否创建供应商
- `vendor.apiKey`：可选，用户填写的远端认证密钥

**响应（200）**：

```json
{
  "success": true,
  "result": {
    "skillsImported": 3,
    "mcpsImported": 2,
    "vendorCreated": true,
    "vendorName": "AICodeSwitch@192.168.1.100",
    "servicesCreated": 5
  }
}
```

## 5. 数据同步逻辑

### 5.1 Skills 写入

```
对于每个用户勾选的 skill:
  生成新 ID
  写入本地 skills 数据
  计入 imported
```

**注意**：前端已完成重名过滤，后端仍做一次安全校验：
```
if 本地已存在同名(name) skill:
  跳过（防御性编程）
```

### 5.2 MCP 写入

```
对于每个用户勾选的 mcp:
  生成新 ID
  写入本地 mcps 数据
  计入 imported
```

后端同样做防御性重名校验。

### 5.3 供应商创建（可选）

```
如果 vendor.enabled == true:
  创建 Vendor:
  {
    id: generateId(),
    name: `${remoteNode.name}@${remoteNode.ip}`,
    description: `从局域网节点 ${remoteNode.ip}:${remoteNode.port} 同步`,
    apiBaseUrl: `http://${remoteNode.ip}:${remoteNode.port}`,
    services: [从 discover 数据创建所有 services]
  }

  每个 service:
  {
    id: generateId(),
    name: `${originalVendorName} - ${originalService.name}`,
    apiUrl: `http://${remoteNode.ip}:${remoteNode.port}/v1/`,
    apiKey: vendor.apiKey || "",
    sourceType: originalService.sourceType,
    authType: originalService.authType,
    supportedModels: originalService.supportedModels,
    // 其他非敏感字段原样复制
  }
```

**关键点**：
- 所有服务的 `apiUrl` 统一指向远端代理地址
- `apiKey` 使用用户在 Step 4 填写的值（可为空）
- 服务名称前加上原始供应商名称以区分来源

## 6. 后端实现要点

### 6.1 鉴权豁免

在 `main.ts` 的鉴权中间件中，将 `/api/lan/discover` 加入豁免路径：

```typescript
app.use('/api', (req, res, next) => {
  if (req.path.startsWith('/auth/') || req.path === '/lan/discover') {
    next();
  } else {
    authMiddleware(req, res, next);
  }
});
```

`/api/lan/discover` 路由内部需要检查 `config.enableLanDiscovery` 开关状态。

### 6.2 AppConfig 扩展

在 `AppConfig` 类型中新增字段：

```typescript
enableLanDiscovery?: boolean;  // 是否允许局域网发现并拉取配置，默认 false
```

### 6.3 扫描辅助接口

`GET /api/lan/scan` 接口需要获取本机局域网 IP：
- 读取网络接口信息（使用 Node.js `os.networkInterfaces()`）
- 过滤出 IPv4、非内部（非 127.0.0.1）的地址
- 返回 IP 和子网前缀（前三段）

### 6.4 同步写入接口

`POST /api/lan/sync` 按以下顺序执行：
1. 写入 Skills
2. 写入 MCPs
3. 如启用，创建 Vendor + Services
4. 返回同步结果统计

## 7. 前端实现要点

### 7.1 扫描策略

- 从 `/api/lan/scan` 获取本机 IP 和子网前缀
- 生成 IP 列表：`<subnet>.1` ~ `<subnet>.254`，排除本机 IP
- 并发控制：同时最多 30 个请求，使用 `Promise.allSettled` 收集结果
- 单个请求超时：1500ms
- 总扫描预计耗时：254 / 30 × 1.5s ≈ 13s
- 进度更新：每完成一个请求更新进度条

### 7.2 弹窗状态管理

```typescript
interface SyncModalState {
  step: 1 | 2 | 3 | 4 | 5;

  // Step 1
  scanning: boolean;
  scanProgress: { current: number; total: number };
  discoveredNodes: DiscoveredNode[];
  selectedNode: DiscoveredNode | null;
  manualIp: string;
  manualPort: string;

  // Step 2 & 3（从 discover 响应填充）
  remoteSkills: RemoteSkill[];
  selectedSkillIds: Set<string>;
  remoteMcps: RemoteMcp[];
  selectedMcpIds: Set<string>;

  // Step 4
  createVendor: boolean;
  vendorApiKey: string;

  // Step 5
  syncing: boolean;
}
```

### 7.3 重名检测逻辑

前端在选择节点后（Step 1 → Step 2），立即获取本地 Skills 和 MCP 列表进行对比：

```typescript
// 获取本地数据用于重名检测
const localSkills = await api.getSkills();
const localMcps = await api.getMcps();

const localSkillNames = new Set(localSkills.map(s => s.name));
const localMcpNames = new Set(localMcps.map(m => m.name));

// 标记远端数据中的重名项
remoteSkills.forEach(skill => {
  skill.isDuplicate = localSkillNames.has(skill.name);
});
```

### 7.4 手动添加

- Step 1 底部提供 IP + Port 输入框
- 点击"连接"后单独请求该地址的 `/api/lan/discover`
- 成功则加入已发现列表
- 失败则提示连接错误

### 7.5 Loading 与自动关闭

- Step 5 点击"同步"后，`syncing = true`，弹窗内容替换为 Loading 界面
- 同步 API 返回成功后，`setTimeout` 延迟 500ms 自动关闭弹窗
- 同步失败时，`syncing = false`，显示错误信息，弹窗保持打开

## 8. 错误处理

| 场景 | 处理方式 |
|------|---------|
| 扫描未发现任何节点 | 显示空状态，引导手动输入 |
| 手动连接失败 | 提示"无法连接，请检查 IP 和端口" |
| 远端节点开关已关闭 | 不会出现在扫描结果中（接口返回 404） |
| Skills/MCP 列表为空 | 对应步骤显示"远端无可用数据"，允许直接下一步 |
| 同步写入失败 | 停留在 Step 5，显示错误信息，弹窗不关闭 |
| 同步过程中关闭弹窗 | 二次确认"同步正在进行中，确定关闭？" |
| 局域网 IP 获取失败 | 回退使用 127.0.0.1，提示手动输入 |
| 全部未选择（Skills+MCP+供应商都未勾选） | Step 5 提示"未选择任何同步内容"，同步按钮 disabled |

## 9. 安全考虑

1. **敏感数据隔离**：远端 API Key、上游 API URL 均不传输
2. **主动暴露**：只有主动开启开关的节点才会被发现
3. **网络范围**：仅限局域网，不涉及公网传输
4. **只读拉取**：同步是单向拉取，不会修改远端数据
5. **404 伪装**：开关关闭时返回 404，不暴露节点存在信息
6. **防御性校验**：后端同步接口仍做重名校验，防止前端绕过
